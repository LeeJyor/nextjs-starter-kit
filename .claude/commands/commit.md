변경된 파일을 분석해 컨벤셔널 커밋을 자동 생성합니다. 추가 인자가 있으면 커밋 메시지 힌트로 활용합니다: $ARGUMENTS

다음 순서로 작업하세요:

1. **변경 사항 파악**:
   - `git status`로 스테이징 여부를 확인합니다
   - `git diff HEAD`로 전체 변경 내용을 확인합니다

2. **커밋 타입 결정** (컨벤셔널 커밋 규칙):
   - `feat`: 새 기능, 새 컴포넌트, 새 페이지
   - `fix`: 버그 수정
   - `style`: CSS/Tailwind 스타일 변경
   - `refactor`: 기능 변화 없는 코드 개선
   - `chore`: 설정 파일, 패키지, 환경 변경
   - `docs`: README 등 문서 변경

3. **커밋 메시지 작성**:
   - 형식: `타입: 한국어 요약 (50자 이내)`
   - 변경 파일이 여러 개면 핵심 변경을 중심으로 요약합니다
   - $ARGUMENTS가 있으면 메시지에 반영합니다

4. **커밋 실행**:
   - 변경된 파일을 스테이징합니다 (`git add` — `.env` 등 민감 파일 제외)
   - 작성한 메시지로 커밋합니다
   - 완료 후 `git log --oneline -3`으로 최근 커밋을 보여줍니다
